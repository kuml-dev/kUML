= kuml-core-dsl
:toc:

image:../../docs/images/kuml-logo-simple.png[kUML,width=48,role=left] *kuml-core-dsl*

Language-agnostic DSL builder infrastructure for kUML. Provides the `diagram{}`
entry-point and the `@KumlDsl` scope marker. The concrete element DSLs
(`classOf`, `part`, `softwareSystem`, …) live in the `kuml-metamodel-*` modules.

== Entry Points

[cols="1,3"]
|===
| Function | Description

| `diagram(name, type, block)`
| Creates a `KumlDiagram`. The default type is `DiagramType.CLASS`.
|===

== Usage

[source,kotlin]
----
// *.kuml.kts — no imports needed (provided by KumlScriptDefinition)
diagram(name = "Order System", type = DiagramType.CLASS) {
    // Phase 1: classOf, interfaceOf, association, …
}

// Plain Kotlin file
import io.kuml.core.dsl.diagram
import io.kuml.core.model.DiagramType

val d = diagram(
    name = "Order System",
    type = DiagramType.CLASS,
)
----

Always use *named parameters* — this is a hard convention for all kUML DSL calls.
Named parameters make LLM-generated code unambiguous and improve readability.

== DSL Scope Safety

`@KumlDsl` is applied to all builder classes. This prevents accidentally calling
an outer-scope builder inside an inner lambda:

[source,kotlin]
----
diagram(name = "Test") {
    diagram(name = "Nested") { }  // ← compile error: @DslMarker prevents this
}
----

== Installation

[source,kotlin]
----
dependencies {
    implementation("io.kuml:kuml-core-dsl:0.1.0")
    // kuml-core-model is exposed transitively via api()
}
----
