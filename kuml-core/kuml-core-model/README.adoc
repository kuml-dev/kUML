= kuml-core-model
:toc:

image:../../docs/images/kuml-logo-simple.png[kUML,width=48,role=left] *kuml-core-model*

Pure Kotlin sealed-class metamodel — the root type hierarchy shared by all kUML
modelling languages (UML 2.x, SysML 2, C4).

No EMF, no code generation, no reflection magic.
Just `interface`, `sealed interface`, and `data class`.

== Core Types

[cols="1,3"]
|===
| Type | Description

| `KumlElement`
| Root `interface` for every element in a kUML model.
  Plain `interface` (not sealed) so that language-specific modules can extend it across module boundaries.

| `KumlNamespaceMember`
| Extends `KumlElement` with a `name: String` property.

| `KumlModel`
| Top-level container: one `root` element, a `ModelingLanguage`, a `ModelLevel`, and free-form `metadata`.

| `KumlDiagram`
| A single diagram — `name`, `DiagramType`, and a flat `elements: List<KumlElement>`.

| `DiagramType`
| Enum: `CLASS`, `SEQUENCE`, `STATE`, `COMPONENT`, `USE_CASE` (V1); more in V1.1.

| `ModelingLanguage`
| Enum: `UML`, `SYSML2`, `C4`, `MIXED`.

| `ModelLevel`
| MDA enum: `CIM`, `PIM`, `PSM`, `DEPLOYMENT`.

| `KumlMetaValue`
| Sealed interface for typed metadata values: `Text`, `Integer`, `Decimal`, `Flag`, `Items`, `Entries`.
  All variants are `@Serializable` for use with `kotlinx.serialization`.
|===

== Usage

Direct instantiation is reserved for testing and tooling.
In practice models are built via the DSL in `kuml-core-dsl`:

[source,kotlin]
----
// order.kuml.kts — no imports needed (KumlScriptDefinition provides them)
umlModel("Order Domain") {
    classOf("Order") {
        attribute("id", type = "UUID")
    }
}
----

== Installation

[source,kotlin]
----
dependencies {
    implementation("dev.kuml:kuml-core-model:0.24.2")
}
----
