= kuml-core-ocl
:toc:

image:../../docs/images/kuml-logo-simple.png[kUML,width=48,role=left] *kuml-core-ocl*

OCL (Object Constraint Language) subset interpreter for kUML.
Pure-Kotlin recursive-descent parser and tree-walking evaluator — no ANTLR,
no runtime dependencies beyond the kUML metamodel. GraalVM Native Image compatible.

Validates `constraint(name, body)` DSL declarations attached to `UmlClass` and
`UmlInterface`, used by `kuml-cli`'s `ValidateCommand` and the `kuml-mcp`
`kuml.validate` tool.

== Supported OCL Subset

* Property navigation (`self.attr`, `self.assoc.target`), including cross-metamodel
  navigation for UML, BPMN, and SysML 2 receivers
* Operation-call syntax (`self.attr.toUpper()`, `self.name.substring(1, 3)`) — parsed as
  a dedicated `OperationCall` AST node, dispatched to the standard library first, falling
  back to property-accessor resolution for model-level zero-arg calls
* Collection operations: `size`, `isEmpty`, `notEmpty`, `includes`, `excludes`, `forAll`,
  `exists`, `select`, `reject`, `collect`, `any`, `one`, `isUnique`, `sortedBy`, `sum`,
  `count`, `including`, `excluding`, `union`, `intersection`, `first`, `last`, `asSet`,
  `asSequence`, `closure`, `iterate`
* `String` standard library: `size`, `concat`, `substring` (1-based, inclusive bounds),
  `toUpper`, `toLower`, `indexOf` (1-based, `0` when not found), `isEmpty`, `notEmpty`, `at`
* `Integer`/`Real` standard library: `abs`, `floor`, `round` (half-up, per OMG OCL 2.4
  §7.5.2 — not Kotlin's round-half-to-even), `max`, `min`, `mod`, `div`
* Logical operators: `and`, `or`, `not`, `implies` (`xor` expressible via De Morgan
  combination — no dedicated token)
* Comparison operators: `=`, `<>`, `<`, `<=`, `>`, `>=`
* Arithmetic operators: `+`, `-`, `*`, `/` (always real division), unary `-`
* Integer, Real, String, Boolean, and `null` literals
* Iterator variables (`->forAll(x | x.foo > 0)`)
* `let ... in ...` bindings and `if ... then ... else ... endif`
* OCL type operations: `oclIsTypeOf`, `oclIsKindOf` (walks `UmlGeneralization` chains),
  `oclAsType`, `oclIsUndefined`, `oclIsInvalid`
* `expr@pre` in `post:` constraint bodies (resolves via an explicit pre-state snapshot;
  a no-op without one, since this subset has no stateful operation-invocation runtime)

=== Deliberately out of scope

* Collection-literal syntax (`Set{1, 2, 3}`, `Bag{...}`) — collections are only produced
  via navigation/`collect`, never literal construction
* Tuple types and tuple literals
* `oclType()` reflection
* Message expressions (`^`, `^^`)
* Real operation-invocation semantics for user-defined model operations (call syntax
  parses, but there is no operation-body execution runtime)

=== Conformance

The parser and evaluator are exercised against representative example expressions
manually transcribed from the public *OMG Object Constraint Language (OCL), Version
2.4* specification (formal/2014-02-03) in
`src/test/kotlin/dev/kuml/core/ocl/OclConformanceTest.kt`, including a maintained
feature-coverage matrix. There is no downloadable, machine-readable OMG OCL test-case
package — this is a best-effort conformance check against the published spec examples,
not an official OMG certification.
`src/test/kotlin/dev/kuml/core/ocl/OclBenchmarkTest.kt` anchors evaluation-time
regressions for small and large (10,000-element) collections plus standard-library
string operations.

== Key Types

[cols="1,3"]
|===
| Type | Description

| `OclValidator`
| Top-level entry point. Validates a full `UmlModel` and returns a `KumlValidationResult`.

| `OclParser`
| Recursive-descent parser. Produces an `OclExpression` AST.

| `OclEvaluator`
| Tree-walking interpreter. Evaluates an `OclExpression` against a `self` binding.

| `KumlValidationResult`
| Aggregate result: list of `KumlViolation` entries, `isValid` convenience.

| `KumlViolation`
| Single violation: constraint name, owning classifier, violating instance (if any), message.
|===

== Usage

[source,kotlin]
----
val result: KumlValidationResult = OclValidator.validate(model)
if (!result.isValid) {
    result.violations.forEach { v ->
        System.err.println("[${v.constraintName}] ${v.message}")
    }
}
----
