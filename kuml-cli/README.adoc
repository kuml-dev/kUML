= kuml-cli
:toc:

image:../docs/images/kuml-logo-simple.png[kUML,width=48,role=left] *kuml-cli*

Command-line interface for kUML.

Evaluates `*.kuml.kts` scripts and renders them to SVG or PNG.
Built with link:https://github.com/ajalt/clikt[Clikt 5.1.0].
Distributed as a Fat-JAR via Gradle `application` plugin (`installDist`).

== Subcommands

=== `kuml render`

[source,bash]
----
kuml render order.kuml.kts                         # UML â†’ order.svg
kuml render banking.kuml.kts                       # C4  â†’ banking.svg
kuml render order.kuml.kts --format png            # â†’ order.png
kuml render order.kuml.kts -o /tmp/diagram.svg     # explicit output
kuml render order.kuml.kts --width 1920 --format png
kuml render order.kuml.kts --theme plain           # override the default 'kuml' theme
kuml render stm.kuml.kts --animated --trace run.kuml.trace.v1.json --format mp4
----

Renders once and exits. Auto-detects model type from the script's last
expression â€” UML scripts end with `classDiagram { â€¦ }` / `diagram { â€¦ }`,
C4 scripts end with `c4Model(name = "â€¦") { â€¦ }`, BPMN scripts end with
`bpmnModel(â€¦) { â€¦ }` (the first diagram in the model is rendered). Supports
`svg`, `png` (Apache Batik transcode), `latex`/`tex` (TikZ), `apng`, `webp`,
and `mp4` (H.264/yuv420p, requires `ffmpeg` on `PATH`) output formats. Use
`--animated` (with an optional `--trace` JSON file) to emit SMIL animations
for STATE and ACTIVITY diagrams, and `--layout=grid` to opt into the
experimental pure-Kotlin grid layout engine instead of the ELK default.

=== `kuml watch`

[source,bash]
----
kuml watch order.kuml.kts                          # polls every 500ms, Ctrl+C to stop
kuml watch order.kuml.kts --format png -o out.png
----

Renders immediately on start, then re-renders on every file change (polls `lastModified` every 500 ms).
Script and I/O errors are printed with a timestamp but the watch loop continues â€” it never exits on error.

=== `kuml validate`

[source,bash]
----
kuml validate order.kuml.kts                       # text output, exit 4 on violations
kuml validate order.kuml.kts --output json         # machine-readable JSON
kuml validate order.kuml.kts -o text
----

Evaluates all `constraint(name, body)` declarations attached to `UmlClass` / `UmlInterface` against the model.
Backed by `kuml-core-ocl` (recursive-descent parser, supports navigation, `->size()`, `->forAll`, `->exists`, `->isEmpty`, `->includes`, `implies`, comparison + logic operators).

Exits `4` if any constraint is violated.

=== `kuml fmt`

[source,bash]
----
kuml fmt order.kuml.kts                            # rewrite in place (idempotent)
kuml fmt order.kuml.kts --check                    # exit 5 if formatting would change
kuml fmt *.kuml.kts
----

Canonical text formatter. The formatter is idempotent â€” running it twice yields the same output.
`--check` makes it CI-friendly: exits `5` when reformatting is required, without touching the file.

=== `kuml generate`

[source,bash]
----
kuml generate --plugin kotlin order.kuml.kts                       # â†’ ./generated/
kuml generate --plugin kotlin --package com.example.domain order.kuml.kts
kuml generate --plugin kotlin -o build/generated order.kuml.kts
----

Runs a `KumlCodeGenerator` plugin against the evaluated model. The built-in `kotlin` plugin
(from `kuml-codegen/kuml-gen-kotlin`) emits `data class` for `UmlClass`, `interface` for
`UmlInterface`, and `enum class` for `UmlEnumeration`, respecting multiplicity and inheritance.

=== `kuml import`

[source,bash]
----
kuml import workspace.dsl                          # â†’ workspace.kuml.kts
kuml import workspace.dsl --format structurizr
kuml import workspace.dsl -o src/main/kuml/banking.kuml.kts
----

Parses a diagram file in an external format and emits an equivalent `*.kuml.kts`
script. V1 supports Structurizr DSL only (`--format structurizr`, also the default).

=== `kuml markdown`

[source,bash]
----
# Inline SVG (default) â€” best for static site generators
kuml markdown --input README.md --output README.rendered.md

# Linked SVG â€” write SVGs to disk, reference them via ![](â€¦)
kuml markdown --input docs/intro.md --output docs/intro.out.md \
              --mode linked-svg --assets-dir docs/assets

# Linked PNG with custom width
kuml markdown --input docs/intro.md --output docs/intro.out.md \
              --mode linked-png --assets-dir docs/assets --width 1200
----

Replaces every ```` ```kuml ```` fenced code block in the input Markdown with
a rendered diagram. Three modes: inline SVG, linked SVG asset, or linked PNG
asset. Backed by the `kuml-docs/kuml-markdown` library. See
link:../tools/pandoc/README.adoc[`tools/pandoc/README.adoc`] for a Pandoc
filter alternative.

=== `kuml completion`

[source,bash]
----
kuml completion bash > /etc/bash_completion.d/kuml
kuml completion zsh  > "${fpath[1]}/_kuml"
kuml completion fish > ~/.config/fish/completions/kuml.fish
----

Emits a shell-completion script for `bash`, `zsh`, or `fish` (Clikt built-in `CompletionCommand`).

=== Other subcommands

The CLI has grown well beyond render/watch/validate/fmt/generate/import/markdown/completion.
Additional subcommands, documented in the root link:../README.adoc[README] and the
https://docs.kuml.dev[Handbook]:

[cols="1,3"]
|===
| Command | Purpose

| `dump-json`  | Dump the parsed diagram + model + computed `LayoutResult` as JSON (wasm playground render path). UML, C4, SysML 2, BPMN, Blueprint.
| `serve`      | Start the kUML web UI (live SVG preview, Ktor/Netty).
| `diagnostics`| Emit script compile/eval diagnostics (line/column) as TSV for IDE integration.
| `transform`  | Apply an M2M or M2T transformer to a kUML script.
| `export`     | Export a kUML model to Structurizr DSL, XMI, ARXML, or Eclipse UML2 profile.
| `asciidoc`   | Render `kuml` blocks in AsciiDoc file(s) to SVG or PNG (Antora pre-render step).
| `profile`    | Inspect and validate UML profiles registered in the kUML profile registry.
| `simulate`   | Execute a kUML or SysML 2 state machine / activity against an event sequence.
| `trace`      | Inspect, replay, and export behaviour traces.
| `sandbox`    | Sandbox policy commands â€” validate guards/actions against the sandbox policy.
| `run`        | Run a kUML state machine or activity with a live event adapter (`stdin`/`mcp`/`batch`).
| `reverse`    | Reverse-engineer source code (Java/Kotlin/C#) or ARXML files into a kUML script.
| `chain`      | Inspect an on-chain registered kUML model (connect, verify hash, list events, sign, verify-sig).
| `plugin`     | Manage kUML plugins (install, remove, list, info, init, update, upgrade, search).
| `ai`         | Inspect and manage kUML AI provider configuration, agent tool sets, benchmarks, and pricing.
| `version`    | Show the kUML CLI version, git SHA, JDK, and build time.
| `update`     | Check for and inspect newer kUML releases on GitHub (no self-update).
|===

== Options (render + watch)

[cols="1,1,3"]
|===
| Option | Default | Description

| `-f, --format [svg\|png\|latex\|tex\|apng\|webp\|mp4]`
| `svg` (or from `--output` suffix)
| Output format

| `-o, --output PATH`
| INPUT with adjusted suffix
| Output file path

| `-w, --width INT`
| `1024`
| Width in pixels (PNG only)

| `--theme TEXT`
| `kuml`
| Theme name (`kuml`, `plain`, â€¦). Overrides `kuml.config.kts`.

| `--layout TEXT`
| `auto`
| Layout engine override: `auto` (ELK for all diagram types) or `grid` (experimental `kuml.grid` engine).

| `--animated`
| off
| Emit an animated SMIL SVG (STATE / ACTIVITY diagrams). Optional `--trace` for trace-driven animation.

| `--trace PATH`
| â€”
| Path to a `kuml.trace.v1` JSON file for trace-driven animation (requires `--animated`).

| `--speed FLOAT`
| `1.0`
| Playback speed multiplier for animation (requires `--animated`).

| `--config PATH`
| â€”
| Path to `kuml.config.kts`.
|===

== Exit Codes

[cols="1,3"]
|===
| Code | Meaning

| `0`
| Success (render) or clean Ctrl+C exit (watch)

| `1`
| Usage error (Clikt default)

| `2`
| Script compilation or model evaluation failed (`render` only)

| `3`
| I/O error (`render` only)

| `4`
| OCL constraint violation (`validate` only)

| `5`
| Formatting would change file (`fmt --check` only)
|===

NOTE: `kuml watch` never exits with code 2 or 3 â€” errors are logged and the loop continues.

== Installation

[source,bash]
----
# Build and install locally
./gradlew :kuml-cli:installDist
export PATH="$PWD/kuml-cli/build/install/kuml/bin:$PATH"
kuml render --help
----

For end users, the official channels are:

[source,bash]
----
# Homebrew (macOS / Linux) â€” tap kuml-dev/kuml ships kuml.rb
brew tap kuml-dev/kuml
brew install kuml
----

The Homebrew bottle ships a *self-contained* runtime: the CLI jars plus a
jlink-built Java 21 runtime under `runtime/`. No system JDK is required â€”
the launcher unconditionally points `JAVA_HOME` at the bundled JRE.

Two release artefacts are published on every `v*.*.*` tag:

[cols="1,3"]
|===
| `kuml-<version>.zip`         | distZip â€” needs a Java 21+ on `JAVA_HOME` or `PATH` (smaller, ~80 MB)
| `kuml-runtime-<version>.zip` | bundled image â€” extracts as `kuml-<version>/{bin,lib,runtime}` with the bundled JRE under `runtime/` (~117 MB). What `brew install` consumes.
|===

Library modules are published to Maven Central as `dev.kuml:<module>:<version>`.
The CLI itself is *not* a Maven artefact. See link:../docs/release.md[`docs/release.md`]
for the release workflow details.
