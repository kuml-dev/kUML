= kuml-metamodel-c4
:toc:

image:../../docs/images/kuml-logo-simple.png[kUML,width=48,role=left] *kuml-metamodel-c4*

Pure Kotlin sealed-class hierarchy for the C4 model (Simon Brown) — the metamodel
that backs all V1 C4 diagram types.

All types are `@Serializable` via `kotlinx.serialization`.

== Element Hierarchy

[source]
----
C4Element (sealed)
├── C4Person
├── C4SoftwareSystem
├── C4Container
├── C4Component
├── C4CodeElement
└── C4Relationship

C4Model            top-level container: people, systems, relationships
C4Diagram          a single rendered view (type + selected elements)
C4DiagramType      SYSTEM_CONTEXT, CONTAINER, COMPONENT, DYNAMIC, DEPLOYMENT, LANDSCAPE
----

== DSL Entry Points

All C4 builders are available inside `c4Model { }` lambdas (provided by `kuml-core-dsl`):

[source,kotlin]
----
// no imports needed in *.kuml.kts — c4 DSL is in defaultImports (V1.0+)
c4Model(name = "Internet Banking") {
    val customer = person(name = "Personal Banking Customer") {
        description = "A bank customer"
    }
    val banking = softwareSystem(name = "Internet Banking System") {
        description = "Allows customers to view information and make payments"
    }

    relationship(source = customer, target = banking) {
        technology = "Views account balances and makes payments"
    }

    systemContextDiagram(name = "System Context") {
        include(customer, banking)
    }
}
----
