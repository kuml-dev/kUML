= kuml-codegen-reverse-kotlin
:toc: left
:icons: font

Kotlin source → UML reverse-engineering engine.

Implements `KumlReverseEngine` (id = `"kotlin"`) on top of
`kotlin-compiler-embeddable` PSI (K2 frontend). Discovered at runtime via
`ServiceLoader`. JVM-only — never include in GraalVM Native Image (Kotlin
compiler internals are reflection-heavy).

Introduced in V3.0.8 (Track B).

== Quick Start

[source,kotlin]
----
import dev.kuml.codegen.reverse.ReverseRequest
import dev.kuml.codegen.reverse.registry.ReverseEngineRegistry
import java.nio.file.Path

val engine = ReverseEngineRegistry.byId("kotlin")
    ?: error("Kotlin reverse engine not on classpath")

val result = engine.analyze(
    ReverseRequest(
        sourceRoots = listOf(Path.of("src/main/kotlin")),
        targetModelName = "MyDomain",
    )
)
----

|===

| `data class Foo(...)`             | `UmlClass`                          | `<<data>>`                     | —

| `sealed interface Foo`            | `UmlInterface`                      | `<<sealed>>`                   | —

| `fun foo()`                       | `UmlOperation`                      | —                              | —

| `inline fun foo()`                | `UmlOperation`                      | `<<inline>>`                   | —
| `operator fun foo()`              | `UmlOperation`                      | `<<operator>>`                 | —

== Diagnostic Codes (REV-K-xxx)

[cols="1,1,4"]
|===
| Code | Severity | Meaning

| `REV-K-001` | WARN | No `.kt` files in source root
| `REV-K-002` | WARN | Source root is not a directory
| `REV-K-003` | WARN | File on disk but not in `KotlinCoreEnvironment`
| `REV-K-010` | INFO | Top-level declaration with unknown PSI type skipped
| `REV-K-011` | INFO | Top-level function skipped
| `REV-K-012` | INFO | Top-level property skipped
| `REV-K-013` | INFO | typealias skipped
| `REV-K-020` | INFO | Nested classifier emitted as top-level
| `REV-K-021` | INFO | Anonymous object skipped
| `REV-K-030` | INFO | Enum-entry body discarded
| `REV-K-040` | INFO | Property/parameter without explicit type (inferred)
| `REV-K-050` | INFO | Supertype outside source set — relationship dropped
| `REV-K-099` | INFO | Forwarded kotlinc front-end message
|===

== Architecture Notes

* **Single fresh `KotlinCoreEnvironment` per `analyze()` call**, wrapped in a
  `Disposable`. The session is disposed in `finally`. No state leaks
  between calls.
* **Two passes**: pass 1 builds the FQN pool (`KtFqnPool`); pass 2 maps
  classifiers, then post-passes add generalization/realization and
  associations.
* **PSI-only — no type binding**. `KtTypeResolver` is heuristic and
  resolves a type-reference text to an internal-pool id (or external
  reference) by suffix-matching against the known FQNs.

== Known Limitations

* Type-binding is heuristic — same simple-name across packages picks
  the first FQN match.
* Annotations on Kotlin declarations are not mapped to stereotypes yet
  (placeholder `KtStereotypeApplier`).
* `expect`/`actual` (Kotlin Multiplatform) is JVM-only territory; the
  engine does not split source sets.
